Efficient Graph-Based Image Segmentation

发表于 2019-12-19 | 更新于 2019-12-25 | 分类于 Digital-Image-Process | 评论数：
Efficient Graph-Based Image Segmentation论文翻译理解

简介

从现在开始，我将要对这篇论文的原理和代码进行详细的解读。

Abstract

​ 这篇文章主要解决把图像分割为区域的问题。我们定义一个预测器，用基于图的图像表示方法来测量两个区域之间边界的可信度。基于这个预测器，我们开发了一种高效的分割算法，并且证明，尽管这种算法具有贪心策略，但是它依然满足全局性能。在构建图时，我们用两种不同的局部邻域来分割图像，并且用真实图像和合成图像来说明结果。这种算法的运行时间和图中的边缘成线性关系。这种方法的特性是在忽视高变化区域的细节的同时还能保留低变化区域的细节。

Introduction

为了能够让图像分割像边缘检测一样广泛运用到生活中，图像分割需要具有下面这些属性：

  • 在视觉上捕捉到关键的分组和区域，这些通常影响到图像的全局观念。这里有两个重要的观点就是哪些是重要的特征以及如何用分割技术来识别它们。
  • 更高效，运行时间和像素个数成线性关系。

虽然有其它的方法更高效，但是它们没有捕获到一幅图像非局部的特性。

本论文方法的优点：

  • 捕获到了重要的非局部特征
  • 时间复杂度为$O\left( n\log n \right)$

​ 与某些经典的聚类方法一样，我们的方法基于在图中选择边，其中每一个像素代表图中的一个节点，某些临近的像素被无向边连接，像素之间的差异由边上的权重来衡量。然而，和经典的方法不同，我们的技术基于图像中邻域区域变量的等级动态的调整分割标准。这导致了一种方法，当做出贪婪的决定时，可以显示出服从某些不明显的全局属性。我们还表明，与这里所开发的标准密切相关的其他适应性标准会导致计算困难的问题(NP hard)。

​ 在做图像分割，首先需要理解如何定义一个区域，我们人眼可以很轻松地解决这个问题，但是使用数字化语言定义就没有那么容易了。比较直观的区域划分方法有，区域的颜色，边缘，纹理。我们来看一个例子：

许多人认为左上图由三个区域组成：左半边强度逐渐增大的区域、右半边一个中心有孔的恒定强度的区域和一个位于右半部分恒定强度区域内高变化强度的区域。我们的目的是找到一中分割算法来捕捉这些区域的重要属性。

  • 强度的大范围变化不能用作多区域的分割标准，这些大范围强度变化不仅出现在左边的渐变区，还出现在右边的高强度变化区。因此，不能假定各区域的强度几乎是恒定的或缓慢变化的。

  • 不能使用纯粹的局部决策来获得三个区域，因为在恒定区和斜坡区边界的地方的强度差比高强度变化区的强度变化小。在这里我用局部方差测试了一下见 图像局部方差计算

    结合上面两点，为了分割图像，需要一种自适应、非局部的判别准则。

在3.1节介绍的衡量两个区域边界可信度的方法是通过比较两个性质来实现的：一个基于边界上的强度差异（区域之间），另一个基于每个区域内相邻像素之间的强度差异（区域内）。如果两个区域边界上的强度差异相比至少一个区域内的强度差异较大，则它们在感知上是重要的。本文的算法使用的就是这种思想。上图的其它三分图像是算法处理得到的结果。
